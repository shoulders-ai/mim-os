# workspace contract

Every Mim workspace is a directory with three contract files and a runtime directory. The contract files are committed; the runtime directory is gitignored.

## Contract files

::: rows
- `mim.yaml` -- Project configuration: name and optional sync mode/remote.
- `AGENTS.md` -- the durable system prompt for any agent working in this workspace. Contains `{{TOOL_SET}}`, `{{SKILL_CATALOG}}`, `{{WORKSPACE_TREE}}`, `{{PROJECT_LOG}}`, `{{AGENT_CONTEXT}}`, and `{{DATE_TODAY}}` template variables resolved at runtime.
- `CLAUDE.md` -- contract pointer. The default content is `@AGENTS.md`, which causes CLI agents (Claude Code) to read the same instructions.
:::

`mim init` creates all three files plus the `.mim/` directory if they do not exist, and merges `.mim/` into `.gitignore`.

## mim.yaml schema

The parser (`parseMimYaml`) accepts two top-level keys:

`name` (string, required) -- workspace display name.

`sync` (object, optional) -- explicit workspace sync mode. `mode` is `manual` or `managed`. `remote` is an optional remote name.

Unknown top-level keys are ignored. App availability comes from direct Mim,
Team, and Project directories; skill activation and integration account
defaults are Personal state rather than Project configuration.

## .mim/ layout

The `.mim/` directory is gitignored runtime state:

::: rows
- `.mim/sessions/` -- chat session JSON files.
- `.mim/history/` -- per-file local recovery baselines.
- `.mim/packages/` -- per-app runtime data: `enabled.json` (personal enablement and trust ledger), app data (`data/`), and run records (`runs/`).
- `.mim/team/` -- managed symlink/junction to the Personal Team checkout; shared files live in its writable `files/` directory.
- `.mim/settings.json` -- workspace settings including `automationApprovalMode`.
- `.mim/traces/` -- unified trace stream JSONL day files and payload blobs.
- `.mim/agent-context.md` -- volatile current-state digest regenerated by `workspace.orient`.
- `.mim/log.md` -- optional human-readable activity log.
:::

## AGENTS.md role

`AGENTS.md` is the standing contract for every agent session. It is loaded as the system prompt with template variables resolved. The default scaffold includes workspace layout documentation, tool and skill catalogs, app SDK guidance, workspace rules, and convention notes. You edit it to shape how agents work in your workspace.

`CLAUDE.md` defaults to `@AGENTS.md`, making CLI agents (Claude Code) read the same instructions. You can replace it with workspace-specific CLI agent rules if needed.
